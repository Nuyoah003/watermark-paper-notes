## REMARK-LLM: A Robust and Efficient Watermarking Framework for Generative Large Language Models
Ruisi Zhang, Shehzeen Samarah Hussain, Paarth Neekhara, Farinaz Koushanfar
University of California, San Diego | USENIX Security 2024

### 一、论文概述
本文提出REMARK-LLM，一个针对大语言模型（LLM）生成文本的鲁棒且高效的水印框架。该框架旨在解决LLM生成内容的知识产权保护问题，防止恶意利用（如垃圾邮件、抄袭）。
### 二、研究背景与动机
2.1 问题定义
• LLM生成的文本具有高度的商业价值和知识产权，但容易被恶意利用
• 现有文本水印方法存在以下挑战：
- 稀疏性：文本token数量有限（最多8.2k），相比图像的65000+像素位置，嵌入空间受限
- 敏感性：文本对微小改动极其敏感，可能导致语义失真
- 脆弱性：对手可能通过改写、编辑等攻击移除或破坏水印
2.2 现有方法的局限
• 推理时水印（如KGW）：计算开销大（2×时间，2.1×GPU内存），语义保留差
• 神经网络水印（如AWT）：水印容量有限，8bit时WER下降到~70%，无法支持16bit以上
• 需要一个既能保证语义完整性，又能支持更大水印容量的方案
### 三、核心思路与创新点
3.1 整体框架
REMARK-LLM包含三个核心模块：
• 消息编码模块（Message Encoding）：将二进制签名嵌入到LLM生成的文本中
• 重参数化模块（Reparameterization）：将密集的水印分布转换为稀疏的token分布
• 消息解码模块（Message Decoding）：从水印文本中提取签名
3.2 关键创新
• 端到端联合训练：三个模块共同优化，平衡语义保留和水印提取
• 优化的束搜索算法：在生成水印文本时保证文本连贯性和一致性
• 对抗性训练：在训练中引入文本编辑攻击（删除、添加、替换），增强鲁棒性
### 四、方法论详解
4.1 消息编码模块
输入：原始文本T和二进制签名M
过程：
• 文本T通过Seq2Seq编码器Se获得隐空间表示Se(T)
• 签名M通过线性层Rm和共享归一化层Rn映射到相同隐空间
• 在隐空间中融合：Se(T+M)，得到水印分布S(T+M)
输出：词汇表上的水印分布
4.2 重参数化模块
• 问题：消息编码生成的是密集分布，但需要稀疏的token序列
• 解决方案：使用Gumbel-Softmax方法
- 添加Gumbel噪声（温度τ）到水印分布
- 通过argmax操作获得离散token
- 温度参数控制语义保留和水印强度的权衡
4.3 消息解码模块
• 使用Transformer预测从水印文本中提取的签名
• 支持从被篡改的文本中恢复签名（对抗性训练）
4.4 优化的束搜索算法
• 标准束搜索会破坏文本连贯性
• 改进方案：
- 在每个解码步骤添加Gumbel-Softmax噪声
- 动态调整束宽和EOS终止阈值
- 保证生成的文本既有高水印提取率，又保持语义完整性
### 五、损失函数与训练
5.1 语义损失（Semantic Loss）
• 目标：最小化原始文本T和水印分布S(T+M)之间的差异
• 使用交叉熵损失，支持随机掩码以增强鲁棒性
5.2 消息恢复损失（Message Recovery Loss）
• 目标：最小化插入的签名M和提取的签名M'之间的差异
• 使用L1损失
• 同时考虑正常提取和对抗性变换后的提取
5.3 联合优化
• 总损失：L = w1*Ls + w2*Lm
• w1、w2为权衡系数，默认均为0.5
### 六、实验设置
6.1 数据集
• HC3：ChatGPT生成的QA回答（19440训练，4860测试）
• WikiText-2：Wikipedia文章段落（44800训练，4360测试）
• ChatGPT Abstract：GPT-3.5生成的论文摘要（8000训练，2000测试）
• Human Abstract：人工撰写的论文摘要（8000训练，2000测试）
6.2 评估指标
• 有效性：水印提取率（WER）- 成功提取的水印比例
• 保真度：
- BERT-S：原始文本和水印文本的语义相似度
- BLEU-4：4-gram匹配度，衡量文本连贯性
• 效率：插入/提取时间和GPU内存占用
• 鲁棒性：在各种攻击下的性能（删除、添加、替换、改写）
• 不可检测性：对手无法区分水印文本和原始文本
### 七、实验结果与分析
7.1 短文本（80 tokens）对比
与AWT对比：
• 4-bit：REMARK-LLM WER=97.01% vs AWT 96.32%（相当）
• 8-bit：REMARK-LLM WER=95.59% vs AWT 74.08%（提升21.51%）
• 16-bit：REMARK-LLM WER=73.46% vs AWT 50%（AWT失败）
• 语义保留：REMARK-LLM BERT-S=0.92 vs AWT 0.96（略低但可接受）
与KGW对比：
• KGW虽然WER高（99.43%），但BERT-S仅0.62，BLEU-4接近0
• 说明KGW严重破坏了文本语义
7.2 长文本（640 tokens）对比
• 16-bit：REMARK-LLM WER=98.93% vs AWT 96.12%
• 32-bit：REMARK-LLM WER=97.84% vs AWT 94.51%
• 64-bit：REMARK-LLM WER=95.61% vs AWT 68.42%（提升27.19%）
• 水印强度（z-score）：REMARK-LLM 7.12（640 tokens）→ 25.20（8.2k tokens）
7.3 泛化性（Transferability）
• 在HC3上训练，在ChatGPT Abstract上测试：
- 4-bit：REMARK-LLM WER=96.98% vs AWT 83.81%（提升13.17%）
• 在WikiText-2上训练，在Human Abstract上测试：
- 4-bit：REMARK-LLM WER=96.85% vs AWT 71.39%（提升25.46%）
• 说明REMARK-LLM具有更好的跨数据集泛化能力
7.4 效率对比
• 时间开销：REMARK-LLM < AWT < EXP（14×）< KGW（2×）
• GPU内存：REMARK-LLM ≈ AWT < EXP/KGW（2.1×-4.5×）
• 神经网络方法的计算开销与LLM大小无关，更具可扩展性
### 八、鲁棒性分析
8.1 文本编辑攻击
• 删除攻击：REMARK-LLM AUC=0.90 vs AWT 0.70（提升20%）
• 添加攻击：REMARK-LLM AUC=0.90 vs AWT 0.69（提升21%）
• 替换攻击：REMARK-LLM AUC=0.88 vs AWT 0.65（提升23%）
• 改写攻击：REMARK-LLM AUC=0.85 vs AWT 0.60（提升25%）
8.2 重新水印攻击
• 对手尝试用新水印覆盖原水印
• REMARK-LLM AUC=0.82，表现良好
8.3 不可检测性
• 词频分布分析：水印文本和原始文本的词频分布相近
• 机器学习检测：使用Transformer、BERT-base、BERT-large进行分类
- 准确率：50%（随机猜测水平）
- F1-Score：0
• 结论：对手无法通过统计或ML方法检测水印
### 九、消融研究
9.1 掩码百分比和Gumbel温度
• 掩码30%：WER=89.97%，BERT-S=0.91，BLEU-4=0.31
• 掩码50%（默认）：WER=93.53%，BERT-S=0.91，BLEU-4=0.29
• 掩码70%：WER=94.98%，BERT-S=0.87，BLEU-4=0.17
• 结论：掩码50%、温度0.3是最优平衡点
9.2 损失系数
• w1=0.3, w2=0.7：WER=94.78%，BERT-S=0.89（语义差）
• w1=0.5, w2=0.5（默认）：WER=93.53%，BERT-S=0.91（平衡）
• w1=0.7, w2=0.3：WER=91.76%，BERT-S=0.92（水印弱）
### 十、跨模型泛化
在不同LLM上的性能：
• OPT-2.7B：WER=93.42%，BERT-S=0.91，BLEU-4=0.34
• OpenOrca-7B：WER=93.70%，BERT-S=0.92，BLEU-4=0.35
• LLaMA-2-7B：WER=91.18%，BERT-S=0.91，BLEU-4=0.39
• 结论：REMARK-LLM对不同LLM架构具有良好的适应性
### 十一、主要贡献
1. 提出了一个端到端的神经网络水印框架，支持更大的水印容量（2×相比AWT）
2. 设计了优化的束搜索算法，在保证文本连贯性的同时最大化水印提取率
3. 通过对抗性训练增强了对文本编辑攻击的鲁棒性
4. 实现了高效的计算（时间和内存开销远低于推理时方法）
5. 实现了不可检测的水印（对手无法区分水印文本）
### 十二、局限与未来工作
12.1 当前局限
• 长文本上的BLEU-4分数较低（0.16-0.32），说明文本连贯性仍有改进空间
• 水印容量仍受限于文本长度（最多支持64-bit）
• 需要在特定数据集上训练，跨域泛化能力有限
12.2 未来方向
• 探索更深层的神经网络架构和混合模型
• 集成Transformer等新型模型以增强上下文建模
• 扩展到其他文本生成任务（翻译、摘要等）
• 研究实时部署的优化策略
### 十三、总体评价
优点：
✓ 水印容量大幅提升（2×相比SOTA）
✓ 语义保留效果好（BERT-S > 0.90）
✓ 计算效率高（时间和内存开销小）
✓ 鲁棒性强（对多种攻击有良好防御）
✓ 不可检测（对手无法识别）
✓ 泛化性好（跨数据集、跨模型）
缺点：
✗ 长文本连贯性仍需改进
✗ 需要在特定数据集上训练
✗ 水印容量仍受文本长度限制
总体：这是一篇高质量的安全与隐私保护论文，在LLM生成内容的知识产权保护方面做出了显著贡献。
